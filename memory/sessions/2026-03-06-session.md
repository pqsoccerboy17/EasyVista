## Session 3 -- ~11:30 AM (continued from Session 2)

### Skills Used
- /frontend-design (plan mode) -- planned React Portal fix for tag dropdown z-index bleed-through
- Manual implementation -- applied portal approach, then reverted and applied simpler z-index elevation fix
- Playwright verification -- tested dropdown in dark + light mode (screenshot tool timed out, accessibility snapshot confirmed fix)

### Key Decisions
- **Portal approach abandoned:** createPortal to document.body broke CSS custom property inheritance in Tailwind v4 dark mode. The dropdown rendered with transparent/wrong background.
- **Z-index elevation chosen instead:** When a dropdown is open, the parent task row gets `relative z-30`, creating a stacking context that puts the row (and its absolute dropdown at `z-40`) above all sibling rows. Minimal change, no side effects.
- **FilterBar left alone:** Its dropdown has `z-20` but sits above the task list in a different DOM section, so no layering conflict.

### Actions Taken
- Modified `src/components/task-manager/TaskItem.jsx` -- added conditional `relative z-30` on task row when dropdown open, bumped dropdown z-index from z-20 to z-40, removed stale `backdrop-blur-none` and `isolation: isolate` styles
- Committed (502fe99) and pushed to main -- Vercel auto-deploy triggered

### Corrections

The lesson: always verify with actual screenshots (not just DOM snapshots) before declaring a visual fix done. Playwright accessibility snapshots can't detect z-index bleed-through.
